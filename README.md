O algoritmo de Dijkstra encontrou como menor caminho o fluxo:

Lead → Cadastro → Triagem → Consulta → Confirmacao

O custo total desse caminho foi igual a 8.

Outro caminho possível seria:

Lead → Contato → Consulta → Confirmacao

Porém, esse fluxo possui custo total igual a 12, sendo menos eficiente.

O caminho escolhido pelo algoritmo é considerado mais eficiente porque possui menor custo operacional dentro do CRM. Isso significa menos tempo de processamento, menos etapas críticas e melhor fluxo para levar o lead até a confirmação final.
